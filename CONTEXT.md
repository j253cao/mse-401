## Key Patterns & Conventions

### Frontend

- **State:** `RecommendationsContext` holds `programCode`, `incomingLevel`, `completedCourses`, `termSummaries`, `studentProfile`, `recommendedCourses`
- **Profile persistence:** `useStoredProfile` hook + localStorage key `uw-guide-profile`
- **API:** All calls go through `src/services/api.ts`; base URL from `VITE_API_URL` (default `http://localhost:8000`)
- **Styling:** Tailwind + `cn()` from `@/lib/utils` (clsx + tailwind-merge)
- **Components:** Radix UI primitives in `components/ui/`; path alias `@/` for `src/`
- **Course codes:** Uppercase, no space (e.g. `MATH115`), to match backend

### Backend

- **Endpoints:** `/` (health), `/recommend` (POST), `/resume-recommend` (POST), `/transcript-parse` (POST), `/random-course` (GET)
- **Recommendation:** `get_recommendations()` in `recommender/main.py`; uses cosine similarity on embeddings
- **Env:** `.env` loaded from project root; `load_dotenv` in `api/main.py`

### Data

- **Program core courses:** Generated by `scripts/extract_program_core_courses.py` → `data/degree_requirements/` and `frontend/src/data/`
- **Engineering programs:** Hardcoded in `constants/engineeringPrograms.ts` (14 programs: AE, BME, CHE, etc.)
- **Incoming levels:** `1A | 1B | 2A | 2B | 3A | 3B | 4A | 4B`

## Constraints & Standards

1. **DRY:** Shared types in `frontend/src/types/api.ts`; API logic in `api.ts` service
2. **Type safety:** Use TypeScript interfaces; Pydantic models for backend request/response
3. **Error handling:** `ApiError` class in `api.ts`; backend returns `{ error: string }` on parse failures
4. **CORS:** Backend allows all origins (`allow_origins=["*"]`) for dev
5. **Course format:** Uppercase, no space (e.g. `MATH115`, `CS136`)

## Running the Project

```bash

```

```bash

```
